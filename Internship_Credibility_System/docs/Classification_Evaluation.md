# End-to-End Classification & Evaluation Process
*How the System Analyzes and Scores Input Data*

When a user submits job details, the system runs a rapid, multi-stage evaluation pipeline to determine if the post is a **Scam** or **Legit**. Here is the exact, end-to-end evaluation workflow:

---

## Stage 1: Data Normalization (Preprocessing)
Before the AI can evaluate the text, it must be cleaned of noise:
1. **HTML & Symbol Stripping**: Regex removes raw HTML tags `<p>`, line breaks, and trailing spaces from the `job_description`.
2. **Data Fusion**: The isolated inputs (`company_name`, `email`, `salary`, `job_description`) are merged into a single standardized string using brackets:
   `"[COMPANY] Acme [EMAIL] hr@acme.com [SALARY] Unpaid [DESC] We are looking for..."`
   *This ensures the evaluation engine has all the context in one place.*

---

## Stage 2: Structural Classification (The Heuristic NLP Model)
The system evaluates the *physical structure* and *grammar* of the text, independent of its meaning. It starts with a base Scam Probability of `0.20` (20%) and evaluates three factors:

1. **The Length Penalty**: 
   - Evaluation: Legitimate jobs are typically highly detailed. Short jobs are highly suspicious.
   - Action: If the text is `< 150` characters long, probability increases by `+0.55`. If massive (`> 1200`), probability drops by `-0.12`.
2. **The Formatting Penalty**: 
   - Evaluation: Scammers frequently use excessive uppercase letters to simulate urgency.
   - Action: If the text is more than 12% UPPERCASE, probability increases by `+0.40`.
3. **The Blank Metadata Penalty**:
   - Evaluation: If the `[COMPANY]` tag was left blank, the system flags it as a "generic copy-paste template."
   - Action: Probability increases by `+0.45`.

*Evaluation Output:* The result is a mathematically clamped baseline score between `0.02` (Safe) and `0.98` (Scam).

---

## Stage 3: Semantic Detection (The Rule-Based Extortion Engine)
While Stage 2 checks grammar, Stage 3 actively scans the actual *meaning* and *promises* contained in the text.

1. **The Extortion Dictionary Eval**: 
   - The engine sweeps the text against an array of known malicious keywords (e.g., *"mandatory training"*, *"pay a fee"*, *"₹2,499"*, *"western union"*).
2. **The Phishing URL Eval**:
   - The engine extracts all web links and isolates their core domains. If it finds link shorteners (e.g., `bit.ly` or `tinyurl.com`), it flags them as phishing attempts.
3. **The Score Calculator**:
   - For every keyword or bad link found, the system tallies a `suspicious_flag`. 
   - It calculates a severe multiplier penalty: `min(0.35 * suspicious_flags, 0.95)` and generates exact text Explanations/Reasons for the user.

*Evaluation Output:* A Penalty Fraction (e.g., `0.0` or `0.70`) and an array of textual Extortion Reasons.

---

## Stage 4: The Final Fusion Math (Classification)
The FastAPI controller takes the two separate evaluations and fuses them: 
`Final Probability = Base NLP Score + Syntactic Extortion Penalty`

1. **The "Clean Record" Multiplier**: 
   If Stage 3 found zero extortion flags, the engine multiplies the final probability by `0.6`. This aggressively drops the probability, confirming a **Safe/Legit** classification (e.g., `< 20% Scam`).
2. **The "Malicious Intent" Override**:
   If Stage 3 found *any* extortion flags, the engine immediately overrides whatever Stage 2 said, forcing the final probability to mathematically lock above `0.85`. This guarantees a **High-Risk Scam** classification, instantly overriding any clever formatting a scammer might have used.

---

## Stage 5: Explainable UI Rendering
1. **The Math Delivery**: The backend sends the Final Confidence Percentage (e.g., "92% Scam") to the frontend.
2. **Visual Translation**: The JavaScript engine spins the circular UI loader to match `92%` and turns the entire panel Red.
3. **The Highlight Reel**: The UI reads the Extortion Reasons generated by Stage 3 and actively highlights those exact malicious keywords (like "training fee") inside the user's text box—proving exactly *why* it was classified as a scam!
